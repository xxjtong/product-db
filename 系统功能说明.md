# 产品数据库 — 系统功能说明

## 项目概述

物联网产品中心数据库，支持产品选型对比、规格书生成、方案设计、报价单生成。

**技术栈**: FastAPI + SQLAlchemy 2.0 + Pydantic | Vue 3 + TypeScript + Vite | Docker

**本地环境**:
- 后端 API: http://localhost:8000 (Swagger: /docs)
- 前端: http://localhost:5173
- 账号: admin / admin (DEV_MODE 自动创建)

## 3. AI 方案助手

### 匹配流程 (2026-06 重构)

```
用户输入 → Round 0: LLM(deepseek-chat) 从 db_ctx 直接匹配产品ID
         → JSON {keywords, matches: {关键词: [产品ID]}}
         → 代码验证ID → DB取真实数据 → 去重交替 → 展示
         → DSML fallback: 解析原生tool-call格式 → SQL LIKE
         → Round 1: Chat LLM(deepseek-v4-flash) 流式文本回复
```

### 匹配规则
- 每关键词 top 10 → 跨关键词去重 → 每关键词最多 5 条 → 交错合并
- 价格/排序过滤由 SQL 层处理
- 语义理解: "地下车库"→光照/防尘, "无线"→WiFi/LoRaWAN
- DB Context: ~98KB (414产品[ID:xxx]描述+specs), 300s TTL

### 工具 (4个)
| 工具 | 功能 |
|------|------|
| `search_products` | 关键词+品类+通讯+协议+供电搜索, SQL LIKE 回退 |
| `get_product_detail` | 完整产品详情 |
| `list_categories` | 品类列表 |
| `create_quotation` | 从方案创建报价单 |

### GenUI 动态组件
| 组件 | 触发条件 | 功能 |
|------|---------|------|
| `SolutionProductCard` | 搜索到产品 | 勾选 + 数量 + 加入方案 + 对比 |
| `QuoteDraftCard` | 创建报价单 | 摘要卡片 + 查看按钮 |

### SSE 事件流
```
connect → tool(可见状态) → products → component(GenUI) → text(逐字) → done → quick_replies
```

## 4. 方案 (Solution) 完整流程

```
① 客户信息 ─────────────────────────────────────
   客户名称 / 项目名称 / 状态 / 备注 / 总价

② AI 方案助手 ──────────────────────────────────
   "10个温湿度传感器 + 1个LoRaWAN网关"
   → 🤖 AI推荐产品 (GenUI卡片)
   → ☑勾选产品 + 数量 → [加入方案(N)] [对比N]
   → tool 状态消息实时可见

③ 产品清单 (BOM) ────────────────────────────────
   [+ 批量选品] 弹窗搜索 + 多选 + 批量添加 (懒加载)
   列表: 产品/型号/功能描述/数量/单价/折扣%/小计/备注/删除
   行内编辑: 数量/折扣/备注直接改, 本地更新不重新加载

④ 生成报价单 ───────────────────────────────────
   [生成报价单] → 自动创建 → 跳转报价详情页
   报价单页可编辑BOM表格 (HTML模式, 添加/删除行)
```

## 5. 报价单 BOM 编辑器

- HTML 表格内联编辑, 列: `# | 产品名称 | 型号/SKU | 功能描述 | 数量 | 单价 | 折扣% | 小计 | 备注`
- 按钮: 重新加载/保存/添加行/导出xlsx
- 每行垃圾桶删除 + confirm确认
- 保存回写 `quotation_items` 表 (数据源: 报价单自身)
- XLSX 导出含合计行, 列格式匹配编辑器
- 通知统一用全局 toast (非 alert)

---

## 6. 权限系统 (2026-06 新增)

- **8 张表**加 `created_by` → `users.id`: products, categories, manufacturers, suppliers, 4 dict tables
- `filter_by_ownership()`: 列表查询过滤 (admin 全量, 普通用户 NULL/自己/admin(id=1))
- `check_ownership()`: 单资源 PUT/DELETE 403 校验
- SECRET_KEY 启动校验 ≥32 字符 (DEV_MODE 除外)

## 9. 数据模型核心表

**字典层 (5)**: manufacturers, dict_comm_methods, dict_comm_protocols, dict_power_supplies, dict_sensor_metrics
(均含 `created_by`)

**核心层 (3)**: device_categories, category_spec_definitions, products
(products.category_id → `ondelete=RESTRICT`)

**映射层 (5)**: product_comm_methods, product_comm_protocols, product_power_supplies, product_hardware_interfaces, product_sensor_capabilities

**多对多 (1)**: product_categories (无ORM模型, 统一通过 `product_category_helper.py` 操作)

**业务层 (8)**: suppliers, solutions, solution_items, quotations, quotation_items, product_dependencies, bom_templates, solution_bom_snapshots

**系统层 (7)**: users, login_logs, system_settings, field_settings, ai_conversations, ai_messages, ai_usage_logs, download_logs, product_files
